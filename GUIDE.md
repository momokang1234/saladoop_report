# 🚀 Apps Script 배포 및 권한 설정 가이드

본 시스템의 안정적인 운영을 위해 아래 가이드를 따라 구글 앱스 스크립트(GAS)를 설정해 주세요.

## 1. 초기 권한 승인 (가장 중요)
구글 스크립트가 사용자의 구글 드라이브와 스프레드시트에 접근할 수 있도록 권한을 승인해야 합니다.

1.  **에디터 실행:** 구글 앱스 스크립트 에디터를 엽니다.
2.  **함수 선택:** 상단 도구 모음에 `doPost` 또는 아무 함수나 선택되어 있는지 확인합니다.
3.  **실행 버튼:** [▷ 실행] 버튼을 클릭합니다.
4.  **권한 검토:** "권한 승인 필요" 팝업이 뜨면 `권한 검토`를 클릭합니다.
5.  **계정 선택:** 본인의 구글 계정을 선택합니다.
6.  **고급 설정:** "Google에서 확인하지 않은 앱입니다"라는 화면이 나오면 왼쪽 하단의 `고급`을 클릭합니다.
7.  **이동:** `프로젝트명(으)로 이동(안전하지 않음)`을 클릭합니다.
8.  **허용:** 모든 요청 사항에 대해 `허용`을 클릭합니다.

## 2. 웹 앱 배포 설정
외부(React 앱)에서 데이터를 받을 수 있도록 웹 앱으로 배포해야 합니다.

1.  **배포 시작:** 우측 상단 `배포` -> `새 배포`를 클릭합니다.
2.  **유형 선택:** 좌측 톱니바퀴 아이콘을 눌러 `웹 앱`을 선택합니다.
3.  **설정 값:**
    *   **설명:** `Daily Report System v1` (자유롭게 입력)
    *   **다음 사용자로 실행:** `나` (본인 이메일)
    *   **액세스 권한이 있는 사용자:** **[모든 사용자 (Anyone)]** (매우 중요!)
4.  **배포 완료:** 배포 버튼을 누르고 생성된 **웹 앱 URL**을 복사하여 React 소스코드의 `APPS_SCRIPT_URL`에 붙여넣습니다.

## 3. 코드 수정 후 업데이트 방법
코드를 수정한 후에 그냥 저장만 하면 반영되지 않습니다.

1.  **배포 관리:** `배포` -> `배포 관리`를 클릭합니다.
2.  **수정:** 현재 활성화된 배포 항목의 `연필 아이콘(수정)`을 클릭합니다.
3.  **버전:** 버전 드롭다운에서 **[새 버전]**을 선택합니다.
4.  **다시 배포:** 하단의 `배포` 버튼을 클릭합니다. (URL은 동일하게 유지됩니다.)

## 4. 구글 드라이브 폴더 설정
사진 저장 시 에러를 방지하기 위해 저장용 폴더의 링크 공유 설정을 확인하세요.

1.  사진이 저장될 폴더를 우클릭합니다.
2.  `공유` -> `링크가 있는 모든 사용자`에게 `뷰어` 또는 `편집자` 권한을 부여합니다. (시트에서 바로 사진을 보기 위해 필요합니다.)
